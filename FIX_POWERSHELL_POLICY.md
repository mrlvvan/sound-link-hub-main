# Решение проблемы с политикой выполнения PowerShell

## Проблема
```
npm : Невозможно загрузить файл ... так как выполнение сценариев отключено в этой системе
```

## Решение 1: Изменить политику выполнения (рекомендуется)

### Шаг 1: Откройте PowerShell от имени администратора

1. Нажмите `Win + X`
2. Выберите "Windows PowerShell (администратор)" или "Терминал (администратор)"

### Шаг 2: Измените политику выполнения

Выполните команду:

```powershell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

На вопрос "Выполнить эту команду?" ответьте `Y` (Yes) или `A` (Yes to All).

### Шаг 3: Проверьте что политика изменилась

```powershell
Get-ExecutionPolicy
```

Должно отобразиться: `RemoteSigned`

### Шаг 4: Закройте и откройте новый терминал

Закройте текущий терминал и откройте новый (не обязательно от администратора).

### Шаг 5: Теперь npm должен работать

```powershell
cd C:\Users\mrlvv\Desktop\sound-link-hub-main
npm install
npm run dev
```

## Решение 2: Использовать командную строку (CMD) вместо PowerShell

Если не хотите менять политику PowerShell:

1. Откройте **Командную строку** (CMD):
   - Нажмите `Win + R`
   - Введите `cmd` и нажмите Enter

2. Выполните команды:

```cmd
cd C:\Users\mrlvv\Desktop\sound-link-hub-main
npm install
npm run dev
```

CMD не имеет таких ограничений на выполнение скриптов.

## Решение 3: Использовать WSL (Ubuntu)

Если предпочитаете использовать Linux окружение:

```powershell
wsl
```

Затем в Ubuntu:

```bash
cd /mnt/c/Users/mrlvv/Desktop/sound-link-hub-main

npm run dev
```

---

## Решение 4: Временно обойти политику (только для текущей сессии)

Если не хотите менять политику навсегда:

```powershell
# Временно измените политику только для текущей сессии
Set-ExecutionPolicy -ExecutionPolicy Bypass -Scope Process

# Теперь npm должен работать
npm install
npm run dev
```

**Примечание:** После закрытия терминала политика вернется к исходной.

---

## Что означает RemoteSigned?

`RemoteSigned` означает:
- ✅ Локальные скрипты (на вашем компьютере) выполняются без подписи
- ✅ Удаленные скрипты (из интернета) требуют цифровой подписи
- ✅ Это безопасная настройка для разработки
