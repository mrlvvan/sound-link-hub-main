# Что делать после запуска Docker контейнера

## Шаг 1: Проверка статуса контейнера

```bash
# Проверьте что контейнер запущен
docker ps
```

Вы должны увидеть контейнер `sound-link-hub` в списке запущенных контейнеров.

## Шаг 2: Просмотр логов

```bash
# Просмотр логов в реальном времени
docker compose logs -f

# Или только последние логи
docker compose logs --tail=50
```

Если видите ошибки, остановите просмотр логов нажав `Ctrl+C`.

## Шаг 3: Открытие приложения в браузере

Приложение должно быть доступно по адресу:

**http://localhost:8080**

Откройте этот адрес в браузере Windows (Chrome, Edge, Firefox и т.д.).

## Шаг 4: Если приложение не открывается

### Проверьте что контейнер работает:

```bash
# Статус контейнера
docker ps

# Детальная информация
docker compose ps
```

### Проверьте логи на ошибки:

```bash
docker compose logs sound-link-hub
```

### Проверьте что порт открыт:

```bash
# Проверка порта в WSL
netstat -tuln | grep 8080
```

### Перезапустите контейнер:

```bash
# Остановка
docker compose stop

# Запуск
docker compose start

# Или полный перезапуск
docker compose restart
```

## Шаг 5: Полезные команды для управления

```bash
# Остановка контейнера
docker compose stop

# Запуск остановленного контейнера
docker compose start

# Перезапуск контейнера
docker compose restart

# Просмотр использования ресурсов
docker stats

# Остановка и удаление контейнера
docker compose down

# Остановка и удаление с очисткой образов
docker compose down --rmi all
```

## Шаг 6: Вход в контейнер (для отладки)

Если нужно что-то проверить внутри контейнера:

```bash
docker exec -it sound-link-hub sh
```

Внутри контейнера вы можете:
- Проверить файлы: `ls -la /usr/share/nginx/html`
- Проверить конфигурацию nginx: `cat /etc/nginx/conf.d/default.conf`
- Выйти: `exit`

## Готово!

Если все работает, приложение доступно по адресу **http://localhost:8080**
