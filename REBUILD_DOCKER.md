# Как пересобрать Docker контейнер с новыми изменениями

## Проблема
Изменения в коде не видны, потому что Docker контейнер был собран со старым кодом.

## Решение: Пересобрать контейнер

### Шаг 1: Откройте Ubuntu (WSL)

Не используйте PowerShell Windows! Откройте Ubuntu через:
- Меню Пуск → Ubuntu
- Или в PowerShell: `wsl`

### Шаг 2: Перейдите в директорию проекта

```bash
# Если проект на рабочем столе Windows
cd /mnt/c/Users/mrlvv/Desktop/sound-link-hub-main

# Или если скопировали в WSL
cd ~/sound-link-hub-main
```

### Шаг 3: Остановите и удалите старый контейнер

```bash
docker compose down
```

### Шаг 4: Пересоберите и запустите с новым кодом

```bash
docker compose up -d --build
```

Флаг `--build` пересоберет образ с новым кодом.

### Шаг 5: Проверьте что контейнер запущен

```bash
docker ps
```

### Шаг 6: Откройте в браузере

Откройте **http://localhost:8080** в браузере Windows.

Теперь вы должны увидеть:
- ✅ Боковую навигацию слева (на десктопе)
- ✅ Адаптивные сетки вместо горизонтальной прокрутки
- ✅ Увеличенные контейнеры на больших экранах
- ✅ Нижняя навигация скрыта на десктопе

## Если изменения все еще не видны

### Очистите кэш браузера

Нажмите `Ctrl + Shift + R` (или `Ctrl + F5`) для жесткой перезагрузки страницы.

### Проверьте логи сборки

```bash
docker compose build --no-cache 2>&1 | tail -50
```

### Полная пересборка без кэша

```bash
docker compose down
docker compose build --no-cache
docker compose up -d
```
